Hexo博客搭建速记

Author: Henry Li   Date: 2025-08-27   Category: 技术速记    本文阅读量: 次

1. 简介

快速、简洁且高效的博客框架；官网：https://hexo.io/，https://hexo.io/zh-cn/

  1. 超快速度： Hexo是基于node.js的快速生成静态博客的静态开源框架;
  2. 编辑简单： 支持Markdown格式和大多数Octopress插件;
  3. 快速部署： 一个指令即可部署到Github Pages、Giteee，Heroku等，强大的API可无限扩展，拥有数百个主题和插件;
  4. 插件和可拓展： 强大的 API 带来无限的可能，轻易集成数种模板引擎（EJS，Pug，Nunjucks）和工具（Babel，PostCSS，Less/Sass）。

需要安装npm和github，并下载hexo脚手架

2.3 目录介绍

文件/目录 作用说明
_config.yml 博客的全局配置文件，比如网站标题、作者、语言、部署方式、主题名等。
package.json 记录你安装的依赖（如主题、插件），可理解为“项目说明书”。
scaffolds 存放文章模板，比如 post.md、page.md，你新建文章时会套用这里的模板。
source/_posts 你写的博客文章都放在这里，文件名就是文章链接的一部分。
source/_drafts 草稿文章，运行 hexo server –draft 可本地预览，但不会发布。
themes 所有主题都放在这里，比如 themes/butterfly、themes/next 等。每个主题有自己的 _config.yml 和模板文件。

3. 主题

博客默认主题太丑了，谁不想把自己的博客打扮的漂漂亮亮的呢

如果你不需要修改默认主题，可跳过本章节(>_<)

Hexo主题：https://hexo.io/themes/

操作指南：https://hexo.io/zh-cn/docs/themes

3.1 详解

修改主题需要从Hexo主题选择自己喜欢的主题。 并修改 _config.yml 内的 theme 设定

对应的每个主题包都会有以下结构

路径 作用 常见修改点
_config.yml 主题级配置：颜色、布局开关、社交链接、第三方 CDN、评论系统参数等 90% 的“主题美化/功能”都改这里
languages YAML/JSON 键值对，如 zh-CN.yml： 自定义文案、添加新语言、修正翻译
layout 按页面类型分文件： 换 DOM 结构、加广告位、调整侧边栏顺序
scripts .js 文件会在 hexo g 或 hexo s 时被 Node 执行，可： 写自定义插件但不想单独发包时放这里
source 目录结构会被完整拷贝到最终站点的根目录： 换字体、覆盖 CSS、加自定义 JS

4. 博客文章

4.1 模版

添加的文章会根据./scaffolds目录下的模版进行创建初始信息

4.2 添加页面

如果点击tag、category、about页面可能会显示报错，则需要自行添加tag、category、about页面，也可自行添加自己的页面

添加的页面会根据./scaffolds/page.md模版创建

4.3 添加文章

需要发布的文章可以主要是Markdown文件，可以用以下三种方式创建

方式一：直接创建并发布

这种方式会创建文章为./source/_posts/草稿标题.md文件夹下创建文件，创建后可直接前端访问到页面

方式二：先创建草稿，再发布

这种方式会创建文章为./source/_drafts/草稿标题.md，创建后可直接前端不可访问到页面

需要进行“发布”后，前端才可访问

方式三：直接拖放文件

直接使用任何软件编辑编辑md（Markdown）文件，放到./source/_posts/文件夹或./source/_drafts/文件夹下

2. 设置令牌

如果源码仓库为私有，则需要创建个人访问令牌，否则跳过

在 Hexo 项目仓库 执行 Github Actions 向 username.github.io 仓库推送代码，由于 Github 权限限制，我们需要在 GitHub 账户中创建一个具有足够权限的个人访问令牌（Personal Access Token，简称 PAT）。这个令牌需要有足够的权限来修改仓库。

4. 推送代码到远程仓库

进行正常推送流程，把源码推送到GitHub远程私有仓库，推送后会自动触发工作流.github/workflows/pages.yml把生成的项目部署到你的页面
